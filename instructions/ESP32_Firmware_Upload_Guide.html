Solar Soil IoT

Instructions for uploading firmware to Sensor Node and Gateway Node boards

Overview

Two ESP32 boards need firmware:

BoardFileRole
Sensor Nodelora/node.inoReads sensors, sends via LoRa, deep sleeps
Gateway Nodelora/gateway.inoReceives LoRa, forwards to MQTT over WiFi

3. Wiring — Gateway Node (gateway.ino)

Same LoRa pins as node above. No extra sensors — WiFi handles internet.

4. Configure Before Uploading

node.ino

Line 38 — Change frequency if needed:

  if (!LoRa.begin(868E6))   // 868 MHz for Europe/India
  if (!LoRa.begin(915E6))   // 915 MHz for US/Americas

gateway.ino

Line 5-6 — Set your WiFi credentials:

const char* ssid = "YourWiFiName";
const char* password = "YourWiFiPassword";

Line 18 — Match LoRa frequency with node.

5. Upload Steps

  1. Sensor Node: Open lora/node.ino in Arduino IDE
  2. Tools → Board → ESP32 Arduino → select your ESP32 model
  3. Tools → Port → select the COM port
  4. Click Upload (→ button)
  5. Open Serial Monitor (Tools → Serial Monitor, 115200 baud) to verify
  6. Gateway Node: Repeat with lora/gateway.ino

6. Verification

Node Serial Monitor output:

Sending packet: {"temp":28,"humidity":65,"soil":42,"v":5.2,"current":410}
Going to sleep...

Wakes every 15 minutes to send again.

Gateway Serial Monitor output:

Connected to MQTT Broker
Forwarding to MQTT: {"temp":28,"humidity":65,"soil":42,"v":5.2,"current":410}

7. Troubleshooting

ProblemFix
LoRa init failedCheck wiring, ensure frequency matches between node & gateway
DHT sensor failedCheck GPIO 4 wiring, try DHT11 instead of DHT22
WiFi not connectingVerify SSID/password, ensure 2.4 GHz band
MQTT not connectingGateway needs internet access via WiFi
No data on dashboardRun lora/simulator.py to test without hardware
